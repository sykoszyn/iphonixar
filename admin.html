⚙️ Configuración de Supabase

Ejecutá estos SQLs una sola vez para habilitar todas las funciones

✅ PASO 1 — Habilitar borrado de productos

Copia este SQL y ejecútalo en Supabase → SQL Editor ↗

-- Función que borra con permisos de superusuario (bypasa RLS)
CREATE OR REPLACE FUNCTION borrar_producto(pid uuid)
RETURNS void
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public
AS $$
BEGIN
  DELETE FROM productos WHERE id = pid;
END;
$$;

-- Dar permiso de ejecutar a la clave pública
GRANT EXECUTE ON FUNCTION borrar_producto(uuid) TO anon;
GRANT EXECUTE ON FUNCTION borrar_producto(uuid) TO authenticated;

📷 PASO 2 — Crear bucket para fotos permanentes

Las fotos se guardarán en Supabase Storage (permanentes, sin límite de tamaño en la DB)

-- Crear bucket público para fotos de productos
INSERT INTO storage.buckets (id, name, public, file_size_limit, allowed_mime_types)
VALUES (
  'fotos-productos',
  'fotos-productos',
  true,
  10485760,  -- 10MB máximo por archivo
  ARRAY['image/jpeg','image/png','image/webp','image/gif']
)
ON CONFLICT (id) DO NOTHING;

-- Política: cualquiera puede ver las fotos (público)
CREATE POLICY "fotos_public_read" ON storage.objects
  FOR SELECT USING (bucket_id = 'fotos-productos');

-- Política: la clave anon puede subir fotos
CREATE POLICY "fotos_anon_upload" ON storage.objects
  FOR INSERT WITH CHECK (bucket_id = 'fotos-productos');

-- Política: la clave anon puede borrar sus fotos
CREATE POLICY "fotos_anon_delete" ON storage.objects
  FOR DELETE USING (bucket_id = 'fotos-productos');

✅ Después de ejecutar los dos SQLs:

• El botón Borrar funcionará correctamente
• Las fotos se guardarán permanentemente en Storage
• No hay límite de tamaño ni las fotos desaparecerán
